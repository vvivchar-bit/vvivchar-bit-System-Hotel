# Питання 15. Організація роботи з вимогами

## Питання

**Організація роботи з вимогами.**

## Відповідь

Організація роботи з вимогами — це впорядкований процес, під час якого вимоги до програмної системи збираються, формулюються, класифікуються, уточнюються, реалізуються та перевіряються.

Вимоги не повинні існувати окремо від програми. Кожна важлива вимога має бути пов’язана з конкретною функцією системи, елементом інтерфейсу, дією користувача або результатом роботи програми. Якщо вимога не має реалізації або доказу виконання, її складно перевірити.

У проєкті **«Програмна система “Готель”»** робота з вимогами організована через зв’язок між навчальним завданням, функціями системи та готовою реалізацією. Для цього в системі створено сторінку **«Покриття вимог реалізацією»**, де кожна вимога має свій ID, формулювання, опис реалізації, доказ для скріну та статус виконання.

## Як організовано роботу з вимогами в проєкті «Готель»

У проєкті робота з вимогами виконується послідовно.

Спочатку визначається основна мета системи: забезпечити перегляд номерів, створення бронювання та керування заявками в готелі.

Після цього визначаються користувачі системи. У проєкті є дві основні ролі: клієнт і адміністратор. Клієнт переглядає номери та створює бронювання, а адміністратор переглядає заявки й керує ними.

Далі формулюються основні вимоги. Наприклад, система повинна відображати список номерів, дозволяти створити бронювання, перевіряти обов’язкові поля, зберігати заявки, показувати адміністративну панель і давати можливість змінювати статуси бронювань.

Після формулювання вимоги пов’язуються з конкретними частинами програми. Наприклад, вимога до перегляду номерів реалізована на головній сторінці, вимога до створення бронювання — на сторінці бронювання, а вимога до керування заявками — в адміністративній панелі.

Завершальним етапом є перевірка виконання вимог. Для цього кожна вимога має доказ: скрін сторінки, повідомлення системи, таблицю заявок, статус або інший результат роботи програми.

## Реалізація в програмній системі «Готель»

У системі **«Готель»** організація роботи з вимогами реалізована через сторінку **«Покриття вимог реалізацією»**.

На цій сторінці вимоги подані не хаотично, а структуровано. Кожна вимога має окремий ідентифікатор, наприклад `FR-01`, `FR-02`, `FR-03`, `NFR-01`, `DR-01`. Це дозволяє швидко знайти потрібну вимогу та відрізнити її від інших.

Для кожної вимоги вказано її формулювання. Це потрібно для того, щоб було зрозуміло, що саме повинна виконувати система.

Також для кожної вимоги вказано реалізацію у програмі. Наприклад, вимога до створення бронювання реалізована через форму бронювання, а вимога до перегляду заявок — через адміністративну панель.

Окремо зазначено доказ для скріну. Це показує, яким саме способом можна перевірити виконання вимоги.

У кінці для кожної вимоги вказано статус. Якщо вимога виконана, вона має статус **«Перекрито»**. Це означає, що вимога не просто описана, а реалізована в програмі та має підтвердження.

## Приклад організації вимог у проєкті

| Елемент організації | Приклад у системі «Готель»                                                         |
| ------------------- | ---------------------------------------------------------------------------------- |
| ID вимоги           | `FR-01`, `FR-02`, `FR-03`, `NFR-01`, `DR-01`                                       |
| Формулювання        | Система повинна відображати список номерів, створювати бронювання, перевіряти поля |
| Реалізація          | Головна сторінка, форма бронювання, адміністративна панель                         |
| Доказ               | Скрін головної сторінки, форми, адмін-панелі або повідомлення                      |
| Статус              | Перекрито                                                                          |

Така організація дозволяє контролювати роботу з вимогами на всіх етапах: від формулювання до перевірки реалізації.

## Підтвердження реалізації

Для цього питання використовується один основний доказ — скрін сторінки **«Покриття вимог реалізацією»**, тому що саме він показує, як у проєкті організовано роботу з вимогами.

## Висновок

Отже, у проєкті **«Програмна система “Готель”»** робота з вимогами організована структуровано. Спочатку визначаються мета системи, користувачі та основні функції. Потім вимоги формулюються, класифікуються, пов’язуються з конкретними частинами програми та перевіряються через докази реалізації.

Сторінка **«Покриття вимог реалізацією»** показує, що кожна основна вимога має ID, формулювання, реалізацію, доказ і статус. Це дозволяє контролювати виконання вимог і доводить, що система розроблялася не випадково, а відповідно до визначених вимог.
