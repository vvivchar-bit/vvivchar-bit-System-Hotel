# UML-діаграми станів програмної системи «Готель»

Документ містить опис UML-діаграм станів, створених для навчальної програмної системи «Готель».

## 1. Тема та мета

**Тема:** Розробка UML-діаграм розроблюваної системи: діаграми станів та діаграми активності.

**Мета:** навчитися будувати UML-діаграми станів та діаграми активності для розроблюваної програмної системи.

У межах роботи розглядається програмна система «Готель», яка призначена для пошуку номерів, бронювання, скасування бронювання та перегляду станів номерів.

## 2. Створені UML-діаграми

Для програмної системи «Готель» було створено три UML-діаграми станів:

1. Діаграма станів — Стан номера.
2. Діаграма станів — Стан бронювання.
3. Діаграма станів — Пошук номера.

Діаграми створено у програмі **Umbrello UML Modeller**.

## 3. Діаграма станів «Стан номера»

Діаграма станів «Стан номера» показує, у яких станах може перебувати номер готелю під час роботи системи.

Основні стани номера:

- **Вільний** — номер доступний для бронювання.
- **Заброньований** — номер уже зарезервований клієнтом.
- **Зайнятий** — номер зайнятий клієнтом.
- **Недоступний** — номер тимчасово недоступний через технічну проблему.

Основні переходи між станами:

- Вільний → Заброньований — створення бронювання.
- Заброньований → Зайнятий — заселення клієнта.
- Зайнятий → Вільний — виїзд клієнта.
- Заброньований → Вільний — скасування бронювання.
- Вільний → Недоступний — технічна проблема.
- Недоступний → Вільний — відновлення номера.

## 4. Діаграма станів «Стан бронювання»

Діаграма станів «Стан бронювання» описує життєвий цикл бронювання в системі «Готель».

Основні стани бронювання:

- **Нове бронювання** — створена нова заявка на бронювання.
- **Очікує перевірки** — система перевіряє введені дані.
- **Підтверджене** — бронювання успішно підтверджене.
- **Помилка бронювання** — під час перевірки виявлено помилку.
- **Скасоване** — бронювання скасоване клієнтом.
- **Завершене** — бронювання завершене після виїзду клієнта.

Основні переходи між станами:

- Нове бронювання → Очікує перевірки — перевірка даних.
- Очікує перевірки → Підтверджене — дані коректні.
- Очікує перевірки → Помилка бронювання — помилка даних.
- Підтверджене → Завершене — виїзд клієнта.
- Підтверджене → Скасоване — скасування клієнтом.
- Помилка бронювання → Скасоване — скасування некоректної заявки.

## 5. Діаграма станів «Пошук номера»

Діаграма станів «Пошук номера» показує процес пошуку номера в системі.

Основні стани пошуку:

- **Очікування запиту** — система очікує введення параметрів пошуку.
- **Введення параметрів пошуку** — користувач вводить тип або інші параметри номера.
- **Обробка запиту** — система перевіряє список номерів.
- **Результати знайдено** — система знайшла номери, що відповідають запиту.
- **Результатів не знайдено** — система не знайшла відповідних номерів.
- **Відображення результатів** — система показує користувачу результат пошуку.

Основні переходи між станами:

- Очікування запиту → Введення параметрів пошуку — початок пошуку.
- Введення параметрів пошуку → Обробка запиту — введено тип номера.
- Обробка запиту → Результати знайдено — збіги знайдено.
- Обробка запиту → Результатів не знайдено — збігів немає.
- Результати знайдено → Відображення результатів — показ списку.
- Результатів не знайдено → Відображення результатів — показ порожнього результату або повідомлення.

## 6. Висновок

У межах роботи було створено UML-діаграми станів для програмної системи «Готель».

Діаграма «Стан номера» показує життєвий цикл номера готелю. Діаграма «Стан бронювання» описує процес переходу бронювання від створення до підтвердження, скасування або завершення. Діаграма «Пошук номера» демонструє логіку обробки пошукового запиту.

Створені діаграми допомагають краще зрозуміти поведінку системи та можуть використовуватися для подальшого проєктування і розробки програмного забезпечення.
