# UML-моделювання програмної системи «Готель»

Документ містить опис UML-діаграм, створених для розширеного аналізу вимог і моделювання внутрішнього устрою програмної системи «Готель».

## 1. Мета моделювання

Метою UML-моделювання є візуальне представлення внутрішньої структури програмної системи, взаємодії її основних об’єктів, алгоритму бронювання номера та компонентної структури вебзастосунку.

## 2. Створені UML-діаграми

У межах теми 5 було створено такі UML-діаграми:

1. Діаграма класів.
2. Діаграма послідовності.
3. Діаграма діяльності.
4. Діаграма компонентів.

## 3. Діаграма класів

Діаграма класів описує внутрішню структуру програмної системи «Готель».

Основні класи системи:

| Клас | Призначення |
|---|---|
| Клієнт | Зберігає дані клієнта та описує його дії під час бронювання. |
| Номер | Описує готельний номер, його тип, ціну та статус. |
| Бронювання | Зберігає інформацію про бронювання номера. |
| СистемаГотель | Центральний клас, який керує номерами, бронюваннями та збереженням даних. |
| Статистика | Відповідає за підрахунок кількості номерів і бронювань. |
| СервісОплати | Моделює оплату бронювання. |
| EmailСервіс | Моделює надсилання підтвердження бронювання клієнту. |

## 4. Діаграма послідовності

Діаграма послідовності показує взаємодію об’єктів під час бронювання номера.

Основний сценарій:

1. Клієнт вводить дані бронювання.
2. Система перевіряє доступність номера.
3. Номер повертає свій статус.
4. Система перевіряє коректність дат.
5. Створюється бронювання.
6. Змінюється статус номера.
7. Email-сервіс надсилає підтвердження.
8. Клієнт отримує повідомлення про результат.

## 5. Діаграма діяльності

Діаграма діяльності описує алгоритм бронювання номера.

Основні етапи:

1. Введення даних клієнта.
2. Вибір номера.
3. Перевірка доступності номера.
4. Перевірка коректності дат.
5. Створення бронювання.
6. Зміна статусу номера.
7. Збереження даних.
8. Надсилання підтвердження.
9. Показ повідомлення клієнту.

Якщо номер недоступний або дати некоректні, система показує повідомлення про помилку.

## 6. Діаграма компонентів

Діаграма компонентів показує основні частини вебзастосунку.

Компоненти системи:

| Компонент | Призначення |
|---|---|
| GitHub Pages | Публікує вебзастосунок в Інтернеті. |
| HTML-інтерфейс | Відповідає за структуру сторінки. |
| CSS-оформлення | Відповідає за зовнішній вигляд сторінки. |
| JavaScript-логіка | Реалізує пошук, фільтрацію, бронювання, скасування та статистику. |
| LocalStorage | Зберігає дані бронювань у браузері. |
| Документація | Містить матеріали аналізу, глосарій та UML-моделі. |
